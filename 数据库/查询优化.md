# 查询优化

## 数据库如何提升查询效率

### 问题

数据库如何提升查询效率？

### 参考答案

**索引优化**
- 为高频查询字段建立合适的索引（B+树、哈希、全文索引等）
- 使用复合索引覆盖多条件查询，注意最左前缀原则
- 避免索引失效：不在索引列上使用函数、隐式类型转换、前导通配符 LIKE '%xxx'

**SQL 语句优化**
- 只查需要的字段，避免 SELECT *
- 用 EXISTS 替代 IN（子查询数据量大时）
- 避免在 WHERE 中对字段做运算或函数调用
- 合理使用 LIMIT 分页，深分页用游标或延迟关联

**表结构设计**
- 合理范式化与反范式化，减少不必要的 JOIN
- 选择合适的数据类型（越小越好，定长优于变长）
- 大表考虑垂直拆分（冷热分离）或水平分表（sharding）

**查询缓存与读写分离**
- 热点数据用 Redis 等缓存层拦截
- 主从复制，读请求走从库分担压力

**执行计划分析**
- 用 EXPLAIN / EXPLAIN ANALYZE 查看执行计划
- 关注 type（扫描方式）、rows（扫描行数）、Extra（是否 filesort / temporary）

**架构层面**
- 分库分表应对数据量膨胀
- 引入搜索引擎（Elasticsearch）处理复杂检索
- 物化视图 / 汇总表预计算高频聚合查询

### 易错点

- 加了索引不代表一定会用到，需要用 EXPLAIN 验证
- 过多索引会拖慢写入性能
- 深分页 OFFSET 很大时性能急剧下降

### 知识延伸

- 索引的底层数据结构（B+树、哈希表）
- MySQL 慢查询日志分析
- 分库分表中间件（ShardingSphere、Vitess）
