# 发布订阅模式与观察者模式

## 问题

发布订阅模式是什么，观察者模式是什么？

## 主题

设计模式 - 行为型

## 难度

中级

## 参考答案

### 观察者模式 (Observer)

一对多的直接依赖关系：**被观察者（Subject）直接通知观察者（Observer）。**

```js
class Subject {
  constructor() {
    this.observers = [];
  }

  addObserver(observer) {
    this.observers.push(observer);
  }

  notify(data) {
    this.observers.forEach(ob => ob.update(data));
  }
}

const subject = new Subject();
subject.addObserver({ update: (data) => console.log('观察者A收到:', data) });
subject.addObserver({ update: (data) => console.log('观察者B收到:', data) });

subject.notify('状态变了');
```

特点：Subject 知道有哪些 Observer，直接持有它们的引用。

### 发布-订阅模式 (Pub/Sub)

通过一个**事件中心（Event Bus）**解耦发布者和订阅者，双方互不认识。

```js
class EventBus {
  constructor() {
    this.events = {};
  }

  on(event, callback) {
    if (!this.events[event]) this.events[event] = [];
    this.events[event].push(callback);
  }

  off(event, callback) {
    if (!this.events[event]) return;
    this.events[event] = this.events[event].filter(cb => cb !== callback);
  }

  emit(event, data) {
    if (!this.events[event]) return;
    this.events[event].forEach(cb => cb(data));
  }
}

const bus = new EventBus();

// 订阅者 — 不知道谁会发布
bus.on('orderCreated', (order) => console.log('发邮件:', order.id));
bus.on('orderCreated', (order) => console.log('扣库存:', order.id));

// 发布者 — 不知道谁在订阅
bus.emit('orderCreated', { id: 1001 });
```

### 核心区别

```
观察者模式:     Subject  ──直接通知──>  Observer
                (两者互相知道)

发布订阅模式:   Publisher ──> EventBus ──> Subscriber
                (两者完全解耦，通过事件中心中转)
```

| | 观察者模式 | 发布订阅模式 |
|---|---|---|
| 耦合度 | Subject 直接持有 Observer 引用 | 发布者和订阅者完全不知道对方 |
| 中间层 | 无 | 有事件中心 |
| 通信方式 | 直接调用 | 通过事件名间接通信 |
| 典型实现 | DOM 事件、Vue 的响应式 | Node.js EventEmitter、Redux、消息队列 |

### 实际应用

- **观察者** — Vue 的响应式系统：data 是 Subject，组件的 Watcher 是 Observer
- **发布订阅** — Node.js EventEmitter、微服务间的消息队列（Kafka、RabbitMQ）

## 追问链

- 手写一个 EventEmitter（含 on/off/emit/once）
- Vue 2 和 Vue 3 响应式实现的区别（Object.defineProperty vs Proxy）
- 发布订阅模式的内存泄漏问题（忘记 off）
- 如何实现 once（只触发一次就自动取消订阅）

## 易错点

- 把观察者和发布订阅当成同一个模式（关键区别是有没有事件中心做中转）
- Node.js 的 EventEmitter 严格来说是发布订阅，不是纯观察者
- 忘记取消订阅导致内存泄漏

## 知识延伸

- 简单记忆：观察者是面对面喊话，发布订阅是通过广播站传话
- 发布订阅多了一层中间人，换来的是更彻底的解耦
- 响应式编程（RxJS）是观察者模式的高级演化
