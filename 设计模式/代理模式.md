# 代理模式

## 问题

代理模式是什么？

## 主题

设计模式 - 结构型

## 难度

中级

## 参考答案

核心思想：**在访问目标对象之前加一层"中间人"，由代理控制对目标的访问。**

调用方以为自己在操作真实对象，实际上操作的是代理，代理可以在转发请求前后做额外的事情。

```
Client ──> Proxy ──> RealSubject
```

### ES6 Proxy 示例

```js
const user = { name: '张三', age: 18 };

const proxy = new Proxy(user, {
  get(target, key) {
    console.log(`读取了 ${key}`);
    return target[key];
  },
  set(target, key, value) {
    if (key === 'age' && value < 0) {
      throw new Error('年龄不能为负数');
    }
    target[key] = value;
    return true;
  }
});

proxy.name;       // 读取了 name → '张三'
proxy.age = -1;   // Error: 年龄不能为负数
```

### 常见代理类型

#### 1. 缓存代理

```js
function createCachedFetch(fetchFn) {
  const cache = new Map();
  return async (url) => {
    if (cache.has(url)) return cache.get(url);
    const result = await fetchFn(url);
    cache.set(url, result);
    return result;
  };
}
```

#### 2. 虚拟代理（延迟加载）

```js
// 图片懒加载
function proxyImage(src) {
  const img = document.createElement('img');
  img.src = 'loading.gif'; // 先显示占位图

  const realImg = new Image();
  realImg.onload = () => { img.src = src; };
  realImg.src = src;

  return img;
}
```

#### 3. 保护代理（权限控制）

```js
const adminData = { secret: '机密信息', public: '公开信息' };

const protectedData = new Proxy(adminData, {
  get(target, key) {
    if (key === 'secret' && !currentUser.isAdmin) {
      throw new Error('无权访问');
    }
    return target[key];
  }
});
```

#### 4. 日志/监控代理

```js
function withLogging(api) {
  return new Proxy(api, {
    get(target, method) {
      return (...args) => {
        console.log(`调用 ${method}`, args);
        const start = Date.now();
        const result = target[method](...args);
        console.log(`耗时 ${Date.now() - start}ms`);
        return result;
      };
    }
  });
}
```

### 实际应用场景

| 场景 | 代理做了什么 |
|---|---|
| Vue 3 响应式 | 用 Proxy 拦截数据读写，触发依赖收集和更新 |
| Nginx 反向代理 | 客户端请求先到 Nginx，再转发到后端服务 |
| 图片懒加载 | 先加载占位图，进入视口再加载真实图片 |
| API 缓存层 | 拦截请求，有缓存直接返回 |
| 权限校验 | 在访问资源前检查身份 |

### 代理 vs 装饰器

两者都是"包一层"，区别在于意图：

- **代理** — 控制访问（你能不能用、什么时候用）
- **装饰器** — 增强功能（在原有能力上叠加新能力）

## 易错点

- 混淆代理和装饰器（代理控制访问，装饰器增强功能）
- 认为 Proxy 只能拦截 get/set（实际支持 13 种 trap）
- 忽略 Proxy 的性能开销（频繁触发 trap 在热路径上可能有影响）

## 知识延伸

- 一句话总结：代理模式就是给对象找个"经纪人"，所有请求先过经纪人，经纪人决定怎么处理再转交给本人
- ES6 Proxy 是语言级别的代理模式实现
- Proxy 的局限：无法代理 Map/Set 等内置对象的内部槽（internal slots）
