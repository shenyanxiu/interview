# Skills 与 MCP 的核心区别

## 一句话总结

Skills 是 Agent 的"可插拔知识模块"，负责"知道怎么做"；MCP 是 Agent 的"标准化工具接口"，负责"做得到"。两者配合构成完整的 Agent 能力体系。

## 本质区别

### Skills = 知识层

- 本质：指令包（Markdown + 脚本 + 文档）
- 解决：Agent 不知道"怎么做"
- 类比：一本操作手册
- 运行：Agent 读取指令后自己执行
- 状态：无状态，纯知识
- 场景：PR 审查流程、部署检查清单、编码规范

### MCP = 能力层

- 本质：工具协议（JSON-RPC 通信）
- 解决：Agent 做不到"某件事"（缺工具）
- 类比：一把螺丝刀
- 运行：Agent 通过协议调用外部服务
- 状态：有状态，连接外部系统
- 场景：查数据库、调云服务 API、操作文件系统

## 对比总结表

| 维度 | Skills | MCP |
|------|--------|-----|
| 本质 | 知识包（指令+脚本+文档） | 工具协议（让 Agent 调用外部能力） |
| 解决的问题 | Agent 不知道"怎么做" | Agent 做不到"某件事" |
| 包含内容 | Markdown 指令、脚本、模板 | Tool 定义（schema）+ 服务端实现 |
| 运行方式 | Agent 读取指令后自己执行 | Agent 通过协议调用外部服务 |
| 状态 | 无状态 | 有状态 |
| 标准 | Agent Skills 开放标准 | Anthropic 主导的 MCP 协议 |

## 协作关系

Skills 和 MCP 不是替代关系，而是互补的：

- **Skills** 告诉 Agent "做什么、怎么做"（知识层）
- **MCP** 给 Agent "能做到"的工具（能力层）
- **Powers**（Kiro 特有）= Skills + MCP，把工具和使用指南打包在一起

### 举例说明

- **纯 MCP**：Agent 有了查数据库的工具，但不知道你们团队查数据的规范和安全要求
- **纯 Skills**：Agent 知道查数据应该怎么做，但没有执行查询的工具
- **Skills + MCP**：Agent 既有工具，又懂规范，才能正确完成任务
