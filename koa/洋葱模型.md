# Koa 洋葱模型

## 问题

Koa 洋葱模型是什么？

## 主题

Node.js / Koa 框架

## 难度

中级

## 参考答案

### 核心思想

请求像穿过洋葱一样，从外层到内层依次经过每个中间件的"前半部分"，到达最内层后，再从内层到外层依次经过每个中间件的"后半部分"。关键在于 `await next()` 这一行——它把每个中间件分成了"请求阶段"和"响应阶段"。

### 执行流程

```
请求 → M1前 → M2前 → M3前 → M3后 → M2后 → M1后 → 响应
```

代码示例：

```javascript
const Koa = require('koa');
const app = new Koa();

app.use(async (ctx, next) => {
  console.log('1 - 进入');
  await next();
  console.log('1 - 离开');
});

app.use(async (ctx, next) => {
  console.log('2 - 进入');
  await next();
  console.log('2 - 离开');
});

app.use(async (ctx, next) => {
  console.log('3 - 进入');
  await next();
  console.log('3 - 离开');
});

// 输出顺序：
// 1 - 进入
// 2 - 进入
// 3 - 进入
// 3 - 离开
// 2 - 离开
// 1 - 离开
```

### 实现原理

Koa 通过 `koa-compose` 实现洋葱模型，核心是递归组合：

```javascript
function compose(middleware) {
  return function (context) {
    function dispatch(i) {
      const fn = middleware[i];
      if (!fn) return Promise.resolve();
      return Promise.resolve(fn(context, () => dispatch(i + 1)));
    }
    return dispatch(0);
  };
}
```

每个中间件接收 `next` 参数，`next()` 实际上就是调用下一个中间件。`await next()` 会暂停当前中间件，等内层全部执行完再继续。

### 与 Express 的区别

| 特性 | Koa（洋葱模型） | Express |
|------|----------------|---------|
| 中间件执行 | 双向（进入 + 离开） | 单向线性 |
| 异步控制 | 原生 async/await | 回调 |
| 后置逻辑 | `await next()` 之后自然执行 | 需要借助 `res.on('finish')` 等 |

### 实际应用场景

洋葱模型让"环绕逻辑"变得很自然：

```javascript
// 计算请求耗时
app.use(async (ctx, next) => {
  const start = Date.now();
  await next();
  const ms = Date.now() - start;
  ctx.set('X-Response-Time', `${ms}ms`);
});

// 统一错误处理
app.use(async (ctx, next) => {
  try {
    await next();
  } catch (err) {
    ctx.status = err.status || 500;
    ctx.body = { message: err.message };
  }
});
```

## 易错点

- 忘记 `await next()`，导致后续中间件不执行
- 在 `next()` 前面漏写 `await`，导致后置逻辑不等内层完成就执行
- 误以为 Express 也是洋葱模型（Express 是单向线性的）
