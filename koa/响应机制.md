# Koa 是怎么返回内容的

## 问题

Koa 是怎么返回内容的？和 Express 有什么区别？

## 主题

Node.js / Koa 框架

## 难度

中级

## 参考答案

Koa 不是通过调用 `res.send()` / `res.json()` 这类方法返回内容，而是通过**设置 `ctx.body`**，由 Koa 在所有中间件执行完毕后统一处理响应。

### 基本用法

```javascript
app.use(async (ctx) => {
  ctx.body = 'Hello World';          // 字符串
  ctx.body = { name: 'koa' };        // 对象 → 自动 JSON 序列化
  ctx.body = Buffer.from('hi');      // Buffer
  ctx.body = fs.createReadStream('file.txt'); // Stream
  ctx.body = null;                   // 无内容，状态码 204
});
```

### ctx.body 支持的类型及自动行为

| 赋值类型 | Content-Type（自动设置） | 状态码 |
|---------|------------------------|--------|
| `string`（HTML 标签开头） | `text/html` | 200 |
| `string`（普通文本） | `text/plain` | 200 |
| `Object` / `Array` | `application/json` | 200 |
| `Buffer` | `application/octet-stream` | 200 |
| `Stream` | `application/octet-stream` | 200 |
| `null` / `undefined` | — | 204 |

### 底层原理：响应是什么时候真正发出的

关键点：**`ctx.body` 只是赋值，不会立即发送响应**。Koa 的处理流程是：

```
请求进入 → 所有中间件执行完毕 → Koa 内部的 respond() 函数读取 ctx.body → 调用 res.end() 发送响应
```

源码简化版：

```javascript
// koa/lib/application.js
function handleRequest(ctx, fnMiddleware) {
  const res = ctx.res;
  res.statusCode = 404;
  
  return fnMiddleware(ctx).then(() => {
    respond(ctx);
  });
}

function respond(ctx) {
  const body = ctx.body;
  
  if (body === null || body === undefined) {
    ctx.status = 204;
    return res.end();
  }
  
  if (typeof body === 'string') {
    return res.end(body);
  }
  
  if (Buffer.isBuffer(body)) {
    return res.end(body);
  }
  
  if (body instanceof Stream) {
    return body.pipe(res);
  }
  
  // 对象/数组 → JSON 序列化
  const str = JSON.stringify(body);
  res.end(str);
}
```

### 延迟响应的设计好处

#### 后续中间件可以覆盖响应

```javascript
app.use(async (ctx, next) => {
  await next();
  if (ctx.status === 404) {
    ctx.body = { error: 'Not Found' };
  }
});
```

#### 响应可以被多个中间件协作构建

```javascript
app.use(async (ctx, next) => {
  await next();
  ctx.body = { code: 0, data: ctx.body }; // 统一包装
});

app.use(async (ctx) => {
  ctx.body = { name: 'koa' }; // 原始数据
});
// 最终响应：{ code: 0, data: { name: 'koa' } }
```

### 与 Express 的对比

```javascript
// Express：调用 res.send() 立即发送，之后不能再修改
app.get('/', (req, res) => {
  res.json({ name: 'express' }); // 立即发送
  res.json({ name: 'again' });   // 报错：headers already sent
});

// Koa：赋值 ctx.body 不发送，可以反复修改
app.use(async (ctx) => {
  ctx.body = { name: 'first' };
  ctx.body = { name: 'second' }; // 最终响应是这个
});
```

## 易错点

- 以为给 `ctx.body` 赋值就立即发送了响应，实际上要等所有中间件执行完
- 以为 `ctx.body` 只能赋值一次，实际上可以反复覆盖，以最后一次为准
- 忘记 `ctx.body = null` 会返回 204 而不是 200
