# 前端性能优化有哪些方向（从整个链路分析）

## 问题

前端性能优化有哪些方向，从整个链路来分析。

## 主题

性能优化

## 难度

中级

## 参考答案

从用户请求的完整链路来系统分析，每个阶段都有对应的优化手段：

### 1. 网络请求阶段

**DNS 解析**
- DNS 预解析（`dns-prefetch`）
- 减少域名数量，降低解析开销

**TCP 连接**
- 使用 HTTP/2 多路复用，减少连接数
- 开启 `preconnect` 提前建立连接
- 使用 CDN 缩短物理链路

**HTTP 请求/响应**
- 减少请求数：合并资源、雪碧图、内联小资源
- 减小体积：Gzip/Brotli 压缩、Tree Shaking、代码分割
- 缓存策略：强缓存（Cache-Control）+ 协商缓存（ETag）
- 使用 HTTP/2 Server Push 主动推送关键资源

### 2. 资源加载阶段

**关键资源优先**
- 关键 CSS 内联到 `<head>`，非关键 CSS 异步加载
- JS 使用 `defer` / `async` 避免阻塞解析
- 使用 `preload` 提前加载关键资源（字体、首屏图片）

**按需加载**
- 路由懒加载（动态 `import()`）
- 图片懒加载（Intersection Observer）
- 组件级别的按需加载

**资源体积优化**
- 图片：WebP/AVIF 格式、响应式图片（`srcset`）、压缩
- 字体：`font-display: swap`、子集化
- JS：代码分割、动态导入、移除 polyfill（针对现代浏览器）

### 3. 解析与渲染阶段

**减少渲染阻塞**
- CSS 放头部、JS 放底部或加 defer
- 避免 CSS 中 `@import`（串行加载）
- 减少关键渲染路径长度

**DOM 操作优化**
- 减少 DOM 节点数量和嵌套深度
- 批量 DOM 操作（DocumentFragment、虚拟列表）
- 避免强制同步布局（读写分离）

**避免重排重绘**
- 使用 `transform` / `opacity` 做动画（触发合成层，不走布局）
- 使用 `will-change` 提示浏览器提前优化
- 避免频繁读取布局属性（offsetHeight 等）

### 4. 运行时阶段

**JS 执行优化**
- 长任务拆分（`requestIdleCallback`、时间切片）
- Web Worker 处理 CPU 密集计算
- 防抖节流控制高频事件

**内存管理**
- 及时清除定时器、事件监听
- 避免闭包导致的内存泄漏
- 大列表使用虚拟滚动

**框架层面**
- React：`memo`、`useMemo`、`useCallback`、合理的 key
- Vue：`v-once`、`computed`、合理的组件拆分
- 避免不必要的重渲染

### 5. 感知性能优化

- 骨架屏 / Loading 状态，减少白屏感知
- 首屏优先渲染（SSR / SSG）
- 乐观更新（先展示结果，后同步服务端）
- 动画过渡掩盖加载延迟

### 6. 度量与监控

- Core Web Vitals：LCP、FID/INP、CLS
- 使用 Performance API 采集真实用户数据（RUM）
- Lighthouse 做实验室环境审计
- 建立性能预算，CI 中卡性能回归

## 追问链

- 什么是关键渲染路径？如何优化？
- Core Web Vitals 各指标的含义和优化方式？
- HTTP/2 相比 HTTP/1.1 在性能上有哪些改进？
- 如何做性能监控和告警？
- SSR 和 CSR 的性能差异在哪里？

## 易错点

- 过早优化：没有度量就盲目优化，浪费精力
- 缓存策略配置错误导致用户看到旧版本
- 过度使用 `will-change` 反而消耗更多内存
- 懒加载导致首屏关键资源延迟加载
