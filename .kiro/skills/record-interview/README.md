# record-interview

在用户明确要求时，整理当前对话中的面试问答，经用户确认归档方案后生成结构化 Markdown 存入项目目录。

## 功能特性

- 仅在用户明确要求时触发，不主动整理未被要求记录的对话
- 支持三种对话形态：用户提问 AI 回答、AI 出题用户回答、粘贴的外部面试题
- 自动识别主题、难度，抽取参考答案、追问链、易错点和知识延伸
- 自动扫描项目根目录已有的面试笔记目录，沿用或新建
- 按主题分文件，优先追加到已有文件，避免零碎
- 写入前必须获得用户对归档方案（文件夹、文件名、归档去向）的明确确认
- 不记录用户自己的作答，只保留问题、答案和扩展信息

## 执行流程

### 第一阶段：对话内容梳理

1. 从当前对话中提取所有面试相关问答
2. 按问题、主题、难度、参考答案、追问链、易错点、知识延伸结构化抽取
3. 过滤掉用户自己的作答和无关闲聊

### 第二阶段：归档方案设计

1. 扫描项目根目录，检查已有面试笔记目录
2. 按主题分文件，文件名使用小写英文 + 连字符
3. 判断每个问题是新建文件还是追加到已有文件
4. 生成归档清单

### 第三阶段：用户确认

1. 向用户展示目标文件夹、文件名、每个问题的归档去向
2. 等待用户明确确认（"OK"、"确认"、"开始写入"等）
3. 用户要求调整则回到第二阶段修订方案

> 在用户明确确认之前，不会写入任何文件。

### 第四阶段：生成并写入

1. 按 `references/interview-note-template.md` 模板生成内容
2. 新建文件直接写入，已有文件追加到末尾并更新目录
3. 输出写入总结（文件列表、操作类型、新增问题数）

## 使用方式

对 Agent 说以下类似的话即可自动激活：

- "记录这次面试" / "记录面试问答" / "把刚才的问答记下来"
- "保存面试题" / "整理面试记录" / "归档面试内容" / "存一下这次面试"
- "record interview" / "save interview questions" / "archive interview notes"
